Отдача контента из контроллера может происходить следующими способами.

return (string) $content - просто возвращение из контроллера любой строковой информации, которая будет выведена без дизайна в браузер.
Допустим может использоваться при аякс-ответах

return View::withDesign('viewFileNameWithoutExtension',$dataForView, $dataForDesign) - возвращение контента view файла,
имя которого передано первым параметром, с примененным дизайном. Второй параметр - данные для view файла, а третий - данные для файла дизайна

return View::withoutDesign('viewFileNameWithoutExtension') - возвращение контента view файла, имя которого передано первым параметров.
Второй параметр - данные для view файла

Кроме того, данные для дизайна можно указывать отдельно от View::withDesign(...)
View::setDesignParams($dataForDesign);

Сам дизайн задается как
View::setDesign('realty');

Так же можно изменить формат отдачи запроса на Json установкой параметра
App::response()->setFormat(Response::JSON_FORMAT);
Это можно сделать как в экшене контроллере так и в init() методе контроллера.
